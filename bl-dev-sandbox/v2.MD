# BSORv2

## Primitives

| Name            | Structure                                        | Size (Bytes) | Description                                                                                                             |
|:----------------|:-------------------------------------------------|:-------------|:------------------------------------------------------------------------------------------------------------------------|
| **bool**        | `byte`                                           | 1            | Boolean value (`0x00` = false, `0x01` = true).                                                                          |
| **byte**        | `uint8`                                          | 1            | Unsigned 8-bit integer.                                                                                                 |
| **short**       | `int16`                                          | 2            | Signed 16-bit integer (Little-Endian).                                                                                  |
| **int**         | `int32`                                          | 4            | Signed 32-bit integer (Little-Endian).                                                                                  |
| **float**       | `float32`                                        | 4            | 32-bit floating-point (IEEE 754).                                                                                       |
| **string**      | `int32 len`<br>`int32 ptr`                       | 8            | A section-relative offset and exact byte length pointing to a raw **UTF-8** string inside the section's `blob` region.  |
| **utf16string** | `int32 len`<br>`int32 ptr`                       | 8            | A section-relative offset and exact byte length pointing to a raw **UTF-16** string inside the section's `blob` region. |
| **Vector3**     | `float x`<br>`float y`<br>`float z`              | 12           | Represents a 3D point or direction vector in space.                                                                     |
| **Quaternion**  | `float x`<br>`float y`<br>`float z`<br>`float w` | 16           | Represents a 3D rotation orientation.                                                                                   |
| **Pose**        | `Vector3 position`<br>`Quaternion rotation`      | 28           | Combines a `Vector3` and `Quaternion` to fully describe an object's spatial orientation (e.g., HMD, Sabers).            |

## Structure

### Header

```
0x442d3d69                      - int, unique magic number
majorFileVersion                - byte, major file version
minorFileVersion                - byte, minor file version
tableRowsCount                  - short, table rows count
    
HeaderTableRow[..]              - table, a table with file sections and their metadata 
Section[..]                     - blob, packed binary sections with arbitrary data inside
```

### HeaderTableRow

```
sectionId                       - byte[64], fixed-size ASCII string representing section id, 0 is a termination character
sectionOffset                   - int, absolute offset from the file start
sectionSize                     - int, size of the whole section in bytes
itemSize                        - int, size of a single entry if the section is an array
itemCount                       - int, count of entries if the section is an array
firstItemOffset                 - int, relative offset from the section start to the first array entry, used to store local metadata
blobSize                        - int, size of the blob region at the end of the section, used to store strings
```

Builtin sections include:

| Section Name | Section ID (Hex)   | Section ID (ASCII) |
|:-------------|:-------------------|:-------------------|
| **Info**     | 0x000000004F464E49 | "INFO"             |
| **Frames**   | 0x000053454D415246 | "FRAMES"           |
| **Notes**    | 0x0000000045544F4E | "NOTE"             |
| **Walls**    | 0x000000534C4C4157 | "WALLS"            |
| **Heights**  | 0x0053544847494548 | "HEIGHTS"          |
| **Pauses**   | 0x0000534553554150 | "PAUSES"           |

Order doesn't matter, HOWEVER all the builtin sections **must** come before custom sections!
Note that even though you can place sections in any order, it's still recommended to maintain the idiomatic order
like in the table above. It helps to load data faster because sections like Info fit in the
average first block (e.g. 4kb), so we can avoid multiple syscalls.

## Sections

### Info

```
originalVersion                 - short, original replay version before migration (major + minor)
[..]                            - byte[2], padding  
version                         - string, mod version
gameVersion                     - string, game version
    
playerUniqueId                  - string, the way to identify specific user/hardware
    
playerID;                       - string, player platform unique id
playerName;                     - utf16string, player platform name
platform;                       - string, oculus or steam
trackingSytem;                  - string, tracking system type (OpenVR, Oculus, etc.)
hmd;                            - string, headset type (Oculus Quest, Valve Index, etc.)
leftController;                 - string, raw controllers type description (Oculus touch, etc)
rightController;                - string, raw controllers type description (Oculus touch, etc)
    
hash;                           - string, map hash
mode                            - string, game mode (Standard, OneSaber, Lawless, etc.)
difficulty;                     - string, difficulty name (Easy, ExpertPlus, etc)
bpm                             - float, map BPM
njs                             - float, map NJS
songName;                       - utf16string, song name
mapper;                         - utf16string, mapper name
    
score                           - int, total unmodified score
timestamp;                      - int, play start unix timestamp
timestampEnd;                   - int, play end unix timestamp
startTime                       - float, song start time (practice mode start time, 0 for 'play' button)
endTime                         - float, song end time
endType                         - int, song end type (clear, fail, quit, restart, etc)
speed                           - float, song speed (practice mode)
    
multiplayer                     - bool, whether the replay was recorded in multiplayer mode
leftHanded                      - bool, whether the replay is mirrored by X-axis
[..]                            - byte[2], padding
environment                     - string, environment name (The beginning, etc.)
height                          - float, static height
jumpDistance                    - float, note jump distance
modifiers                       - comma separated string, game modifiers (FS, GN, etc.)
leftSaberOffsets                - Pose, offsets of the left saber, ignoring build one in game
rightSaberOffsets               - Pose, offsets of the right saber, ignoring build one in game
roomPosition                    - Vector3, room position
roomRotation                    - float, room rotation around Y axis

Blob[..]                        - byte[..], contains strings
```

### Frames (Array)

```
time                            - float, song time
fps                             - int, player's FPS
headPose                        - Pose
leftSaber                       - Pose
rightSaber                      - Pose
```

### Notes (Array)

```
{                               - NoteId struct
  spawnID                       - int, note index in the beatmap file
  scoringType                   - ScoringType, modified note scoring type (game value + 2)
  lineIndex                     - int, horizontal grid position from left to right (0 to 3)
  lineLayer                     - int, vertical grid position from bottom to top (0 to 2)
  colorType                     - int, note color type (0 = Left/Red, 1 = Right/Blue)
  cutDirection                  - int, note directional indicator arrow direction (0 to 8)
  angleOffset                   - int, additional counter-clockwise angle rotation offset in degrees
}
eventTime                       - float, song time when points from note cut was applyed to the overall score
cutTime                         - float, song time of actual cut event
spawnTime                       - float, spawn time of note from beatmap, approx time when note will be 0.9m in from of a player
eventType                       - int, good = 0, bad = 1, miss = 2, bomb = 3
timeScale                       - float, time scale of a song at the hit time
unityTimeScale                  - float, time scale reported by unity engine
pose                            - Pose, global spatial position and rotation orientation coordinates of the note at collision slice time
{                               - NoteScore struct
  multiplier                    - int
  totalScore                    - int
  fcScore                       - int
  maxScore                      - int
  combo                         - int
  energy                        - int
}
{                               - NoteCutInfo struct
  speedOK                       
  directionOK                   
  saberTypeOK                   
  wasCutTooSoon                 
  saberSpeed                    
  saberDir                      
  saberType                     
  timeDeviation                 
  cutDirDeviation               
  cutPoint                      
  cutNormal                     
  cutDistanceToCenter           
  cutAngle                      
  beforeCutRating               
  afterCutRating                
  cutDistanceSign               - int, directional modifier mapping the mathematical sign of the raw pre-absolute distance offset direction
}
```

ScoringType is game value + 2. Standard values:

```
Normal = 0,
Ignore = 1,
NoScore = 2,
Normal = 3,
SliderHead = 4,
SliderTail = 5,
BurstSliderHead = 6,
BurstSliderElement = 7
```

### Walls (Array)

```
{                               - WallId struct
  spawnID                       - int, wall index in the beatmap file
  lineIndex                     - int, wall line index
  lineLayer                     - int, wall line layer
  width                         - int, wall width
  height                        - int, wall height
  duration                      - float, wall length
}   
inTime                          - float, time when player entered this wall
outTime                         - float, time when player left this wall
spawnTime                       - float, spawn time of wall
inEnergy                        - float, energy when player enterred this wall
outEnergy                       - float, energy when player left the wall
multiplier                      - int, multiplier after the wall hit
```

### Heights (Array)

```
height                          - float, height value
time                            - float, song time
```

### Pauses (Array)

```
rotation                        - float, angle of rotation
time                            - float, time since song started (in seconds)
```

### Optional (Array)

```
{
	rotation                   - float, angle of rotation
	time                       - float, time since song started (in seconds)
}

{
	njs                        - float, new njs value
	time                       - float, time since song started (in seconds)
}
```
